Summary

Water Decisions does not collect, store, or process any personal data about its visitors. There are no analytics, no tracking pixels, no advertising, no contact forms, no accounts, and no cookies set by this website.

What this website does

Water Decisions is a static reference website hosted on GitHub Pages. It serves HTML, CSS, JavaScript, and images. All content is public research material. The site:

  • does not use cookies of any kind;
  • does not run analytics, tracking, or advertising scripts;
  • does not have user accounts, login, or registration;
  • does not include contact forms or comment systems;
  • does not share any data with third parties for marketing purposes.

Hosting and server logs

This website is hosted on GitHub Pages. As with any web server, GitHub records standard access logs including IP addresses, timestamps, and requested URLs. These logs are managed by GitHub, Inc. under its own privacy policy and are not accessed or used by Visioning Lab Ltd.

See the GitHub General Privacy Statement for details on how GitHub handles this data.

Third-party resources

To keep this site lightweight and fast, a small number of open-source libraries are loaded from public content delivery networks (CDNs). These CDNs may log requests in the same way as any web server. The libraries themselves do not set tracking cookies or transmit personal data:

  • Lucide icons — loaded from unpkg.com for the icons displayed across the site.
  • Chart.js, jsPDF — loaded from cdnjs.cloudflare.com on the Assessment page only, used for radar charts and PDF export. The assessment runs entirely in your browser; no responses are sent anywhere.

If you wish to avoid these CDN requests entirely, you can use a browser with content-blocking enabled.

Your rights

Because Water Decisions does not collect personal data, there is no personal data to access, correct, export, or delete. The UK GDPR rights of access, rectification, erasure, restriction, portability, and objection do not apply to data we do not hold.

If you have any questions about this statement, please open an issue on GitHub.
